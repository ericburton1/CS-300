# CS 300 - Data Structures and Algorithms Portfolio Reflection
In CS 300, I developed an advising assitance program for ABC University that reads course data from a file, validates formatting and prerequisites, stores course objects in an appropriate data structure, and allows advisors to retrieve individual course information or print a full course list in alphanumeric order. The technical challenge was not simply reading and printing data, but evaluating which data structure best satisfied both correctness and performance requirements. I explored three structures-vector, hash table, and binary search tree-and analyzed their runtime and memory characteristics before implementing the final solution in C++. 

My approach began with designing detailed pseudocode for file parsing, validation, and insertion logic. The program required enforcing at least two parameters per line, validating that each prerequisite existed elsewhere in the file, and preventing structural inconsistencies. After building the loaders for vector, hash table, and BST, I performed a runtime analysis of each structure. The vector approach provided straightforward linear insertion but require O(n log n) sorting before printing the course list. The hash table improved average lookup time to O(1), but it did not preserve order, meaning sorted output still required additional processing. The binary search tree provided average-case O(log n) insertion and lookup while naturally supporting sorted output through in-order traversal. Based on the requirement to frequently print courses in alphanumeric order, I selected the BST as the most balanced solution. Througn instructor feedback, I refined my analysis to more clearly distinguish average versus worst-case runtime and to explicitly compare BST tradeoffs against hash table performance and pointer overhead. 

One of the most significant roadblocks during this course was moving beyond "making the program work" and focusing instead on structural correctness and edge-case handling. Implementing removal operations in both the hash table and BST required careful pointer management and special handling for head nodes, two-child removal cases, and memory cleanup. Compiler errors such as missing semicolons and brace mismatches reinforced the importance of deliberate structure and attention to detail. Reviewing error diagnostics and aligning implementation with my pseudocode strenghtened my debugging discipline and deepened my understanding of recursion and pointer logic. Feedback from my BST milestone confirmed that the implementation correctly handled insertion, search, traversal, and removal, including proper use of the in-order successor for two-child removal. 

This project significantly changed how I approach software design. Instead of immediately writing code, I now spend more time analyzing requirements, identifying workload patterns, and evaluating structural tradeoffs. I learned that data structure selection is an architectural decision that directly affects scalability, memory usage, and long-term maintainability. For example, if the workload involved frequent inserts with minimal ordered output, a hash table might be preferable. However, because the advising program loads once and is read frequently for ordered display, the BST provided a more efficient and elegant solution. 

My approach to writing maintainable and adaptable code also evolved. I became more intentional about separating concerns between file parsing, validation, insertion, and search logic. Defensive checks for file errors, prerequisite validation, and duplicate detection improved readability. Consistent naming conventions, modular function design, and explicit destructor logic in the BST enhanced readability and safe memory management. Rather than writing code that simply satisfies the assignment, I now consider how easily another developer could read, modify, or extend the program. 

For my portfolio, I am including the runtime and memory analysis from Project One alongside the working C++ implementation from Project Two that prints the course list in alphanumeric order using in-order traversal. Together, these artifacts demonstrate both theoretical understanding of algorithmic complexity and practical implementation of efficient data structures in C++.
